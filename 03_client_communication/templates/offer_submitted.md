# Template: Offer Submitted

## When to Use
Notifying a buyer that their offer has been submitted. Used at the `offer_pending` stage immediately after the offer goes in. The client should hear from the agent before they start wondering.

## Context to Read
- `address_of_interest` — the property the offer is on
- `offer_price` from `property_object.json` — the amount submitted
- `contact_log` Current State — what the offer terms were (any context the agent captured: contingencies, close date, escalation clause, etc.)
- `timeline` — if the client has a deadline (lease expiring, etc.), acknowledge it

## Subject Line
`Offer Submitted — [Address]`

## Required Content (in order)

1. **Confirmation** — the offer is in. State the address and the offer amount. One sentence.

2. **What happens next** — how long the seller has to respond, what the possible outcomes are (accept, reject, counter). Set expectations so the client is not anxious.

3. **Response timeline** — if known, state when to expect to hear back. If not known, give a realistic range based on standard practice.

4. **What the agent is doing** — one sentence on what the agent is doing while waiting (monitoring, in contact with listing agent, etc.). Clients want to know someone is watching.

5. **Close** — short. Promise to be in touch as soon as there is news.

## Tone Notes
- Match the client's energy without amplifying anxiety. If it is a competitive situation, acknowledge the stakes calmly.
- Do not oversell the offer or predict an outcome. "We put in a strong offer" is fine; "I think they're going to take it" is not.
- Short. The client wants confirmation and a timeline, not a recap.

## Do Not Include
- Offer strategy details that should not be in writing
- Predictions about whether the offer will be accepted
- A long explanation of the offer terms (those were covered when the offer was prepared)
